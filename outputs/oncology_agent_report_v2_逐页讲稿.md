# 《智能体在肿瘤学领域中的应用现状及未来展望》逐页讲稿

说明：以下为可直接照读的汇报文案。实际讲述时可根据会场节奏压缩或展开；所有医学相关表述均建议保持“辅助、证据整理、供 MDT 讨论”的谨慎口径。

## 第 1 页：标题页

建议时间：40秒

各位老师好，我今天汇报的题目是《智能体在肿瘤学领域中的应用现状及未来展望》。这次报告不把重点放在泛泛介绍 ChatGPT，也不讨论 AI 替代医生，而是聚焦 Agent 如何把指南、论文、病历、分子检测和临床试验信息组织成可审阅的证据工作流。接下来我会先讲为什么肿瘤学需要 Agent，再看近年研究做到什么程度，最后用一个低风险的 MDT Evidence Brief 演示概念，说明这类系统应该如何审慎设计和使用。

## 第 2 页：肿瘤学需要的是证据整合型 Agent

建议时间：1分钟

肿瘤学的复杂性不在于单个问题难回答，而在于不同来源的信息需要连续整合。一个患者的诊疗讨论往往同时涉及病历、影像、病理、分子检测、既往治疗、指南、论文、药品标签和临床试验。Agent 的价值不是直接给治疗决定，而是把检索、核对、结构化输出和证据引用连接成流程，把证据链保留下来，供医生和 MDT 审阅。

## 第 3 页：Agent 不是更会聊天，而是能完成流程

建议时间：1分钟

这里我想先区分 LLM、Chatbot 和 Agent。普通 LLM 或 Chatbot 主要是根据输入生成回答，擅长总结、解释和改写，但容易受知识过时和不可核验影响。Agent 则是在 LLM 外面加上任务规划、工具调用、检索、工作流控制和结构化输出。面向医学场景，我更愿意把 Agent 理解为受约束的证据整理流程，而不是更像医生的聊天框。

## 第 4 页：技术底座：把生成能力约束在证据流程里

建议时间：1分15秒

从技术上看，肿瘤学 Agent 通常由几个部件组成。RAG 负责从指南、论文和数据库中检索证据，降低模型只凭记忆回答的风险；Tool Use 负责调用 PubMed、ClinicalTrials.gov、OncoKB 或院内系统；Workflow 把病例整理、问题拆解、证据检索和摘要生成固定成节点；记忆和 EHR 连接用于纵向病程信息；多智能体则可以模拟影像、病理、分子和药物安全等不同角色。但关键原则始终是先检索、再生成，先引用、再解释。

## 第 5 页：应用版图：从检索到多模态协作

建议时间：1分钟

如果把不同应用按临床成熟度和演示安全性放在一张图上，可以看到并不是所有场景都适合先展示。自动治疗规划和多模态诊断目前风险较高，更适合放在未来研究中讨论。相对稳妥的是指南和证据检索、文献追踪、MDT 会前摘要、临床试验初筛和分子报告解释。这些场景可以使用模拟病例和公开资料，输出也更容易做到可引用、可复核。

## 第 6 页：前沿证据显示：能力在进步，临床仍需验证

建议时间：1分30秒

近年的研究给出的信号比较一致：裸 LLM 在精准肿瘤治疗推荐中可靠性仍然不足，但加入检索、工具调用和结构化流程后，表现会明显改善。例如指南检索、临床试验匹配和工具增强的多模态病例分析都显示出潜力。与此同时，系统综述也提示总体准确率和评价体系仍有限，安全性、清晰度、潜在伤害和真实世界泛化还需要进一步验证。因此现阶段更适合把 Agent 定位为 evidence brief、pre-screening 和会前准备工具。

## 第 7 页：场景一：指南与证据检索先落地

建议时间：1分钟

第一个比较适合先落地的场景是指南与证据检索。它的流程是先把病例转成明确的临床问题，再从指南、PubMed、药品标签和知识库中检索证据，然后区分指南推荐、临床试验、回顾性研究和专家意见，最后生成 Evidence Brief。需要强调的是，RAG 不等于事实保证，检索库质量、更新时间、证据适用人群和引用完整性都会影响输出，所以结论必须保留适用条件和人工复核点。

## 第 8 页：场景二：MDT 辅助适合做会前准备

建议时间：1分15秒

第二个场景是 MDT 辅助。MDT 本身就是一个多学科信息整合过程，所以 Agent 最适合做会前准备和会后记录。会前可以帮助标准化病例摘要、生成关键问题清单、提醒缺失检查并进行证据和试验初筛；会中可以快速定位证据来源、记录分歧和不确定性；会后可以辅助结构化归档和随访提醒。这里仍然要强调，它不能替代 MDT 判断，只是提高讨论前的信息密度。

## 第 9 页：场景三：临床试验匹配价值明确

建议时间：1分15秒

第三个场景是临床试验匹配。这个方向的价值比较清晰，因为试验匹配本来就耗时，入排标准复杂，而且信息分散。Agent 可以先抽取患者特征，再检索候选试验，逐条判断 inclusion 和 exclusion，并输出候选列表、匹配理由、排除原因和需要人工核实的内容。TrialGPT 和 TrialMatchAI 这类研究说明了效率提升和检索覆盖的潜力，但最终是否适合入组必须由研究者和临床团队确认。

## 第 10 页：场景四：分子报告解释需要证据等级

建议时间：1分钟

第四个场景是精准肿瘤学中的分子报告解释。这个方向很有价值，但也最容易被过度简化。一个变异不等于一定有药，一个靶点也不等于患者一定适合某种治疗。Agent 应该做的是把肿瘤类型、分期、NGS 或 IHC 结果、既往治疗和合并症放在一起，结合 OncoKB、指南、药品标签和临床试验信息，输出变异解释、证据等级、潜在方向和不确定性，而不是直接形成处方。

## 第 11 页：场景五：多模态 Agent 仍处早期

建议时间：1分钟

第五个方向是多模态 Agent。未来的肿瘤学系统可能会同时整合影像、病理、基因组和文本证据，由一个协调 Agent 组织不同模型的输出。但这仍处于早期阶段。真实临床使用前，需要解决多模态输入质量控制、跨模态冲突处理、外部验证、可解释性、责任归属和网络安全问题。因此我把多模态 Agent 作为未来方向，而不是本次演示的重点。

## 第 12 页：真实边界：不能把研究探索说成临床成熟

建议时间：1分钟

面对严谨的医学听众，边界意识非常重要。我们可以说 Agent 能辅助证据检索、病例摘要、MDT 讨论清单生成和临床试验预筛；但不能说它能替代医生或 MDT 决策，不能说它可以自动诊断或自动治疗规划，也不能保证没有幻觉和偏倚。更合适的口径是：学术演示、临床证据辅助、MDT 信息整理助手。

## 第 13 页：现场演示应选择低风险、高可解释场景

建议时间：1分钟

基于前面的分析，我建议现场演示选择 MDT 证据整合 Agent。它比单纯文献追踪更贴近临床讨论，也比自动治疗规划更安全。输入模拟病例后，系统输出关键问题、证据摘要、临床试验线索和 MDT 讨论清单。这个场景既能连接黑色素瘤会议语境，又不会越过诊疗决策边界。

## 第 14 页：Demo 概念：Onco-MDT Evidence Brief

建议时间：55秒

这个演示应用可以命名为 Onco-MDT Evidence Brief，定位是面向 MDT 会前准备的肿瘤学证据整理助手。左侧输入模拟病例和关键临床问题，右侧输出 Evidence Brief、引用来源、待核实项和免责声明。这里强调两点：第一，不输入真实患者信息；第二，输出只作为学术研究和临床讨论辅助，不作为诊断或治疗建议。

## 第 15 页：概念架构：Dify 可承载证据整理流程

建议时间：1分15秒

在 Dify 这类可视化平台中，可以把这个演示拆成几个节点：病例输入、问题拆解、知识检索、证据筛选和结构化输出。降幻觉设计非常关键：如果没有检索证据，系统必须回答证据不足或需人工核实；输出模板固定为结论、证据、适用条件、限制、不确定项和引用；临床表达使用供讨论，而不是推荐。这样可以把生成能力限制在证据整理流程里。

## 第 16 页：模拟病例：用黑色素瘤连接会议语境

建议时间：1分钟

为了连接本次会议语境，演示可以使用一个黑色素瘤模拟病例。例如：58 岁女性，转移性黑色素瘤，BRAF V600E 阳性，PD-1 单药后进展，ECOG 1。我们不让系统回答下一步怎么治，而是问：下一步 MDT 需要讨论哪些证据与试验方向？这样能体现 Agent 的信息整理价值，同时避免给出直接治疗建议。

## 第 17 页：预期输出：Evidence Brief，而非治疗医嘱

建议时间：1分钟

预期输出应该是 Evidence Brief，而不是治疗医嘱。它包括病例摘要、关键临床问题、证据摘要、临床试验线索、MDT 讨论清单和免责声明。这里的核心表达是：这不是 AI 给方案，而是 AI 帮 MDT 把讨论材料准备得更完整。真实应用前，还需要接入可追溯知识库、检索日志、评价集和医生审核流程。

## 第 18 页：从 Demo 到临床应用还差四层验证

建议时间：1分15秒

从 Demo 到真实临床应用，中间差的不只是工程实现，而是完整验证体系。数据层要处理去标识化、权限、来源质量和偏倚；模型层要验证事实性、引用准确性、鲁棒性和漂移；临床层要有医生评价、真实病例外部验证和流程指标；治理层要明确责任归属、审计日志、监管路径和网络安全。原则上，人类负责最终判断，系统负责可追踪的信息组织。

## 第 19 页：未来展望：可信、可评估、可监管

建议时间：1分钟

未来肿瘤学 Agent 的方向，不是更强的聊天，而是更可靠的临床知识基础设施。它会从单次回答转向工作流，从单模型转向工具链，从平均准确率转向临床指标，从研究原型转向可监管系统。真正有价值的 Oncology Agent，应该成为可审计的证据协作者。

## 第 20 页：总结：Agent 的价值在于人机协同

建议时间：1分钟

最后总结四点。第一，肿瘤学 Agent 的合理定位是证据整合与流程增效。第二，RAG、工具调用和工作流能改善裸 LLM 的可靠性。第三，最适合先落地和演示的是 MDT Evidence Brief 与试验预筛。第四，真实临床应用必须经过验证、审计、监管与人类总控。我的核心观点是：肿瘤学 Agent 不应替代医生，而应帮助医生更高效、更透明地组织证据。
